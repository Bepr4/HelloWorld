# HelloWorld：面向国际局势的多智能体辅助推演平台

## 一、项目定位

HelloWorld 定位为一个面向国际局势、金融市场与复杂现实事件的多 Agent 辅助推演平台。它不是一个单纯的新闻聚合工具，也不是一个追求长期确定性预测的传统沙盘系统，而是在 MiroFish 现有多智能体框架基础上，将真实世界事件、市场变化、预测市场信号、历史金融市场反应和多角色立场分析结合起来，形成一个短周期、多轮反应、重分析判断的智能研判系统。

项目的核心目标，是先构建一套可靠的现实基础信息层，再通过推演启动与事件上下文编排层把基础信息转化为适合多 Agent 读取的上下文材料，然后开展多 Agent 推理，最后由综合输出与前端展示层生成可追溯的综合报告。现实基础信息层主要由新闻 Agent、金融分析 Agent 和事件地图 Agent 三条支线组成：新闻 Agent 负责抓取和整理新闻报道，回答“发生了什么”；金融分析 Agent 负责调用 digital-oracle 等能力分析金融市场信号和历史价格反应，回答“市场如何理解这件事”；事件地图 Agent 负责将前两条支线的产出整理成事件 Context Map，回答“这些材料讲的是什么、如何关联”。三条支线共同产出事件基础信息库和事件 Context Map，再由推演启动与事件上下文编排层进行筛选、组织和分层披露，为后续多 Agent 推理提供材料。

在推理层中，系统让多个 Agent 围绕同一事件进行多角度分析。系统最终输出的不是简单新闻摘要，而是一份能够呈现事件背景、关键变量、新闻事实、金融市场反应、不同立场判断、潜在风险、主要分歧、共识方向和未来可能变化的综合分析报告。

相比直接向 AI 提问一次，本项目更强调“信息底座 + 多主体分析 + 市场信号 + 综合报告”的组合能力。它希望减少单一模型回答的片面性，让复杂事件分析不只依赖模型的静态知识，而是能够结合正在发生的新闻事实、历史关联、金融市场反馈、预测市场概率，以及历史事件发生时的资产价格反应和金融分析材料，为国际形势研究、地缘风险分析、政策影响判断、金融市场观察和投资参考提供更立体、更透明、更可追溯的辅助判断。

## 二、项目要解决的问题

当前直接使用 AI 分析国际局势或市场事件时，常见问题是信息输入分散、分析视角单一、判断依据不透明，并且很难同时把新闻事实、历史背景、金融市场反应、预测市场概率变化和历史事件冲击组织到同一套分析框架中。传统沙盘推演虽然强调多轮互动，但往往更适合长周期设定和角色推演，不一定适合处理每天都在变化的真实新闻事件与短期风险判断。

本项目希望解决的核心问题，是让系统从“给一个问题、得到一个回答”的单点问答模式，升级为“先形成事件基础信息库，再由推演启动与事件上下文编排层把基础信息转化为推理材料和推演计划，调动多类 Agent 进行协同研判，最后由综合输出与前端展示层生成可追溯的综合报告”的分析模式。系统需要能够持续整理现实事件，识别事件之间的关联，把新闻事实、历史背景和金融分析结果沉淀为可引用的信息基础，并在用户提出某个议题、变量或假设后，让不同 Agent 基于各自立场和关注重点进行短周期、多轮反应分析。

因此，项目关注的不是让 AI 断言未来一定会怎样，而是帮助用户看到：一个事件为什么重要，哪些主体会受到影响，不同立场会如何理解它，市场正在释放什么信号，历史上类似事件曾经带来什么市场反应，预测市场的概率变化可能意味着什么，以及当前判断中有哪些风险和不确定性。

## 三、总体框架

HelloWorld 的整体框架可以划分为四大模块：第一层是“现实基础信息层”，第二层是“推演启动与事件上下文编排层”，第三层是“多 Agent 推理层”，第四层是“综合输出与前端展示层”。现实基础信息层负责把外部世界变成结构化、可引用、可追溯的事件材料；推演启动与事件上下文编排层负责决定哪些材料应该被哪些 Agent 读取、以什么粒度读取，并输出结构化推演计划；多 Agent 推理层负责基于这些材料进行多主体、多立场、多轮次的辅助推演；综合输出与前端展示层负责将推演结果整理为综合报告，通过前端工作台展示引用依据，并支持用户继续追问或二次推演。

| 层级 | 核心作用 | 说明 |
| --- | --- | --- |
| 现实基础信息层 | 构建事件基础信息库与 Context Map | 由新闻 Agent、金融分析 Agent 和事件地图 Agent 三条支线组成。新闻 Agent 负责抓取、清洗、聚合现实事件报道；金融分析 Agent 负责调用 digital-oracle，分析预测市场、历史价格、交易信号和金融分析材料；事件地图 Agent 负责将前两类产出整理成事件 Context Map。三类产出共同构成事件基础信息库和事件 Context Map。 |
| 推演启动与事件上下文编排层 | 理解任务、组织上下文、生成推演计划 | 读取事件 Context Map 和事件基础信息库，通过 Planner Agent 理解用户意图、决定 Agent 阵容，将基础信息筛选、索引后转化为不同 Agent 可高效读取的上下文包，并输出结构化 Simulation Plan。 |
| 多 Agent 推理层 | 基于上下文材料进行辅助推演 | 多个具有不同 Profile 的 Agent 读取推演启动与事件上下文编排层提供的上下文包和 Simulation Plan，在新闻事实、历史背景和金融市场证据的支持下，对同一事件进行多角度、短周期、多轮次推理。 |
| 综合输出与前端展示层 | 生成报告、展示过程、支持追问 | 将推演结果落盘为结构化数据，通过 Report Agent 生成包含事件背景、金融市场分析、不同主体判断、分歧、共识、风险和不确定性的综合报告；通过 3D 地球工作台展示事件、推演过程和报告；支持推演后对话、引用溯源和二次推演。 |

## 四、核心能力设计

### 1. 新闻报道采集与事件整理

新闻报道采集与事件整理是现实基础信息层的第一条支线。它的重点不是保存大量孤立新闻，而是把多个来源中关于同一现实事件的报道聚合成统一事件块。一个事件块代表一件现实世界中发生或正在发展的事情，可以包含事件摘要、发生时间、相关地区、相关主体、事件类别、重要程度、可能影响方向和多来源信息概括。

通过这种方式，系统前端展示的内容不再是普通新闻流，而是一个可浏览、可筛选、可选择、可推理的事件面板。用户可以从事件面板中选择某个事件作为分析对象，系统也可以根据用户输入自动检索相关事件，为后续事件基础信息库和多 Agent 推理提供材料。

### 2. 金融分析 Agent

金融分析 Agent 是现实基础信息层的第二条支线。它不直接参与后续多主体立场推演，而是作为一个专门的基础分析 Agent，围绕当前事件调用 digital-oracle 的能力，检索和分析相关金融市场信息。它可以分析当前预测市场定价，也可以回溯历史事件发生时的资产价格反应，还可以检索已有金融分析报告，为事件基础信息库补充“市场如何理解这件事”的证据。

金融分析 Agent 的输出不应只是一个简单结论，而应包括它参考了哪些预测市场、哪些历史价格窗口、哪些交易信号、哪些金融分析材料，以及这些材料分别说明了什么。它的作用是把金融市场数据整理成 Agent 可以引用的结构化市场证据，而不是让后续推理 Agent 自己临时到处查询市场。

### 3. 事件基础信息库

事件基础信息库是连接现实基础信息层与多 Agent 推理层的中间层。它由新闻报道整理结果和金融分析 Agent 输出共同组成，负责保存某个事件在进入推理前所需要的基础材料。

一个事件基础信息库可以包含：事件摘要、时间线、相关主体、相关地区、关键报道、多来源事实对照、历史背景、相似事件、金融市场反应、预测市场定价、历史价格窗口、金融分析报告摘要和引用来源。它既可以进入 RAG 知识库长期沉淀，也可以作为某次多 Agent 推理任务的上下文输入。

通过事件基础信息库，后续推理层不再直接面对零散新闻和原始市场数据，而是面对一份已经整理过、可引用、可追溯的事件材料包。

### 4. 推演启动与事件上下文编排层

推演启动与事件上下文编排层位于事件基础信息库、事件 Context Map 和多 Agent 推理层之间，是本项目中非常关键的中间处理模块。它要解决的问题是：事件基础信息库中的材料可能很多、很杂、粒度不同，不能全部一次性喂给所有推理 Agent；系统需要根据任务目标和 Agent Profile，把基础信息整理成清晰、有限、可引用的上下文包。

这个模块的核心职责包括：从事件基础信息库中提取事件简报，组装本轮任务所需的材料包，根据不同 Agent 的立场和关注点提供差异化信息切片，并在 Agent 需要更多证据时支持进一步追溯原始新闻、金融分析材料、历史事件或市场信号。

因此，它不是简单的 RAG 检索，也不是简单的上下文拼接，而是一个负责“信息筛选、关联组织和渐进式披露”的编排层。它决定哪些材料先给所有 Agent，哪些材料只给特定 Agent，哪些材料作为补充证据按需展开。通过这个模块，多 Agent 推理层可以更高效地读取事件信息，避免被过多原始材料干扰。

### 5. 市场信号分析能力

市场信号分析能力主要由金融分析 Agent 承担，其设计灵感来自 digital-oracle。它的核心作用，是把真实交易数据、预测市场定价、历史价格表现和已有金融分析材料整理成可解释的市场证据，为事件基础信息库补充“市场如何理解这件事”的视角。

对于当前和未来事件，金融分析 Agent 可以读取 Polymarket 等预测市场和相关交易信号，观察市场正在如何给某个结果、风险或政策变化定价。对于历史事件，它需要回到事件发生、公开或升级的时间窗口，观察当时资产价格、波动率、成交量和跨市场联动如何变化，并结合已有金融分析材料解释这些市场反应。

这一层的重点不是把某个 Polymarket 概率、某个价格涨跌或某篇金融分析报告当成最终结论，而是把它们作为可引用、可加权、可交叉验证的市场输入。系统需要区分当前预测市场定价、历史事件冲击、相似案例类比和金融分析报告，避免把不同时间、不同性质的信号混在一起简单投票。

通过这种设计，后续推理 Agent 不需要临时查询零散金融数据，而是可以直接引用金融分析 Agent 已经整理好的市场证据。这样，最终研判就不只是“新闻事实 + 模型推理”，还能够纳入真实资金形成的概率信号、历史市场冲击和风险定价变化。

### 6. Profile 驱动的多 Agent 判断

系统中的推理 Agent 不应是同质化的回答节点，而应通过 Profile 形成不同立场、利益诉求、风险偏好和分析风格。比如国家 Agent 更关注战略利益、外交压力和安全风险，国际组织 Agent 更关注协调空间、规则秩序和冲突降级，媒体或公众视角 Agent 可能更关注叙事框架、社会情绪和传播影响，市场视角 Agent 则可以在金融分析 Agent 提供的基础材料上进一步判断资产价格、风险溢价和投资预期。

同一个事件在不同 Agent 眼中会有不同意义。通过让多个 Agent 同时分析同一事件，系统可以呈现单一回答难以覆盖的分歧和互补视角。用户不仅能看到“综合结论”，也能看到不同主体为什么会形成不同判断。

### 7. 短周期多轮反应分析

本项目不会把重点放在上百轮、长周期、强剧情式的推演上，而是更适合围绕一个现实事件或用户给定变量进行十几轮以内的短周期反应分析。用户可以输入一个事件、假设或提示词，例如某国宣布新的制裁措施、某地爆发新的冲突、某场选举结果发生变化、某个预测市场概率突然上升，或者某个历史事件曾经引发显著市场波动，系统便可以组织多个 Agent 进行多轮反应。

在这个过程中，不同 Agent 可以分别给出事件理解、可能影响、行动倾向、风险判断、对其他主体反应的预判，以及在新信息出现后的判断修正。系统再对这些输出进行汇总，帮助用户看到局势可能如何被不同主体理解和推动。

### 8. 综合输出、前端展示与引用透明

综合输出与前端展示层是本项目的关键产出层。系统在多 Agent 推理完成后，将 Context Map、Simulation Plan、Agent 输出、状态轨迹、风险视图和证据索引落盘为结构化数据，再由 Report Agent 整理、归纳和综合，生成一份结构化辅助判断报告。报告需要呈现事件背景、新闻摘要、市场相关信息、市场信号分析、不同 Agent 的核心判断、主要分歧、共同判断、风险因素、关键不确定性和可能演化方向。

同时，系统应通过前端工作台展示 Agent 判断所依据的信息来源和引用材料，例如它参考了哪些事件块、哪些历史背景、哪些市场指标、哪些历史价格窗口、哪些预测市场问题、哪些交易信号、哪些金融分析材料以及这些信息如何影响了判断。用户既可以阅读最终报告，也可以回看推演过程、追溯引用依据、围绕报告继续追问，或针对某个 Agent 输出进行解释性问答。这样可以提高系统输出的透明度，让用户更容易评估结论的可靠性和参考价值。

## 五、典型运行流程

1. 用户选择某个事件，或输入一个变量、假设、国家、地区、政策、市场问题或预测市场议题。
2. 新闻 Agent 围绕该事件检索和整理相关新闻报道、公告、公开资料和专业报告，将零散信息清洗、聚合为结构化新闻事件块。
3. 金融分析 Agent 调用 digital-oracle，围绕同一事件生成金融市场分析、预测市场信号、历史价格反应和相关金融分析材料。
4. 新闻事件块、原始来源和金融分析结果共同进入事件基础信息库，形成可检索、可引用、可追溯的现实信息底座。
5. 事件地图 Agent 基于事件基础信息库生成事件 Context Map，梳理事件概览、时间线、相关主体、来源差异、事实争议、金融信号和关键不确定性。
6. 推演启动与事件上下文编排层读取事件 Context Map 和事件基础信息库，通过 Planner Agent 理解用户意图、判断任务类型、选择 Agent 阵容，并为不同 Agent 组织公共基础上下文和专项补充材料。
7. Context Orchestrator 输出结构化 Simulation Plan，说明本轮推演要回答的问题、共享哪些材料、启用哪些 Agent、每个 Agent 读取哪些专项材料，以及这些材料对应哪些证据引用。
8. 多 Agent 推理层读取 Simulation Plan，由 Agent Runner 激活常驻视角 Agent 或临时专题 Agent，维护 Simulation State，并执行短周期的状态驱动行动-反应推演。
9. 推演过程中，系统记录每一步的观察、判断、行动提案、裁判结果、状态变化、风险视图和引用依据，形成可回放的推演轨迹。
10. 综合输出与前端展示层将推演产物落盘为结构化 simulation_data，通过 Report Agent 生成综合报告，并在前端工作台中展示事件材料、推演过程、报告章节和引用溯源。
11. 用户可以围绕最终报告继续追问，也可以点开某个 Agent 输出或某个推演步骤，查看其输入材料、判断依据和解释性回答；后续如果输入新变量或新事件，系统可以启动新的推演或二次推演流程。

## 六、项目价值

本项目的价值在于，它把新闻数据库、事件知识库、预测市场数据、历史与实时金融市场信号、金融分析材料和多 Agent 推理组织成一个统一的复杂事件研判框架。它不是为了让系统给出一个单薄的结论，而是帮助用户在信息快速变化、立场高度分化、未来不确定性较强的环境中，获得一个更完整的分析视野。

对于国际形势研究，它可以帮助用户把分散新闻整理成事件脉络；对于地缘风险分析，它可以呈现不同主体可能采取的行动和判断分歧；对于金融市场观察，它可以把政治事件与市场反应联系起来；对于投资参考，它可以结合新闻事实、市场数据和预测概率形成更丰富的风险提示；对于政策和公共事件分析，它可以展示事件可能影响的群体、区域和后续变量。

最终，这个项目希望把 MiroFish 从偏社交互动和推演展示的多 Agent 系统，升级为一个现实信息驱动的多主体辅助判断系统。它的重点不是“预测未来一定会发生什么”，而是“在复杂信息环境下，帮助用户看清哪些信息重要、哪些主体会如何理解、哪些风险正在形成、哪些概率信号值得关注，以及当前判断中还有哪些不确定性”。
